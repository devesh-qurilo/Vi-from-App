# Location API Contract

This frontend expects buyer and vendor location APIs to use the same address shape for GET and PUT responses.

The frontend currently uses:
- `expo-location` for GPS latitude/longitude and reverse geocoded address autofill.
- Dropdown UI for `country`, `state`, `district`, `city`, and `pinCode`.
- A stepped delivery-radius selector for vendors.

Backend should accept values sent from the dropdowns or from `expo-location`. If a GPS-filled value is not already in the frontend starter list, the frontend still sends it as a normal string.

## Shared Address Object

```json
{
  "_id": "address-id",
  "houseNumber": "233",
  "locality": "MG Road",
  "city": "Bengaluru",
  "district": "Bengaluru Urban",
  "state": "Karnataka",
  "country": "India",
  "pinCode": "560001",
  "fullAddress": "233, MG Road, Bengaluru, Bengaluru Urban, Karnataka, India, 560001",
  "latitude": 11.9716,
  "longitude": 22.5946,
  "location": {
    "type": "Point",
    "coordinates": [22.5946, 11.9716]
  },
  "createdAt": "2026-06-09T08:00:43.520Z",
  "updatedAt": "2026-06-09T14:39:52.668Z"
}
```

Important:
- `coordinates` must be `[longitude, latitude]`.
- `latitude` and `longitude` should also be returned as top-level fields for easier frontend use.
- `fullAddress` should be generated by backend if not sent from frontend.
- `country` is currently expected as `India`.
- `state`, `district`, `city`, and `pinCode` should be returned exactly as saved so the dropdown can preselect them.

## India Dropdown Fields

The frontend sends these fields from dropdowns:

```json
{
  "country": "India",
  "state": "Karnataka",
  "district": "Bengaluru Urban",
  "city": "Bengaluru",
  "pinCode": "560001"
}
```

Backend should store these as strings. Do not return nested dropdown objects for these fields.

Good:

```json
{
  "state": "Karnataka",
  "district": "Bengaluru Urban",
  "city": "Bengaluru",
  "pinCode": "560001"
}
```

Avoid:

```json
{
  "state": { "name": "Karnataka" },
  "city": { "name": "Bengaluru" }
}
```

```json
      "houseNumber": "233",
      "locality": "MG Road",
      "city": "Bengaluru",
      "district": "Bengaluru Urban",
      "state": "Karnataka",
      "country": "India",
      "pinCode": "560001",
```

```json
      "deliveryType": "Pickup",
      "deliveryRadius": 0,
      "location": {
        "type": "Point",
        "coordinates": [22.5946, 11.9716]
      }
    },
    "location": {
      "type": "Point",
      "coordinates": [22.5946, 11.9716]
    },
    "latitude": 11.9716,
    "longitude": 22.5946,
    "deliveryType": "Pickup",
    "deliveryRadius": 0
  }
}
```

### PUT `/api/vendor/update-location`

Request body:

```json
{
  "houseNumber": "233",
  "locality": "MG Road",
  "city": "Bengaluru",
  "district": "Bengaluru Urban",
  "state": "Karnataka",
  "country": "India",
  "pinCode": "560001",
  "fullAddress": "233, MG Road, Bengaluru, Bengaluru Urban, Karnataka, India, 560001",
  "latitude": 11.9716,
  "longitude": 22.5946,
  "deliveryRegion": "25km"
}
```

Allowed `deliveryRegion` request values:

```json
["25km", "50km", "75km", "100km", "150km", "200km", "250km", "300km"]
```

Backend can store this as numeric `deliveryRadius`.

Expected response:

```json
{
  "success": true,
  "message": "Vendor location updated successfully",
  "data": {
    "address": {
      "_id": "address-id",
      "vendor": "vendor-id",
      "houseNumber": "233",
      "locality": "MG Road",
      "city": "Bengaluru",
      "district": "Bengaluru Urban",
      "state": "Karnataka",
      "country": "India",
      "pinCode": "560001",
      "fullAddress": "233, MG Road, Bengaluru, Bengaluru Urban, Karnataka, India, 560001",
      "deliveryType": "Pickup",
      "deliveryRadius": 25,
      "location": {
        "type": "Point",
        "coordinates": [22.5946, 11.9716]
      }
    },
    "location": {
      "type": "Point",
      "coordinates": [22.5946, 11.9716]
    },
    "latitude": 11.9716,
    "longitude": 22.5946,
    "deliveryType": "Pickup",
    "deliveryRadius": 25

```

## Validation Rules

- `houseNumber` is required.
- `latitude` is required.
- `longitude` is required.
- For buyer profile location, these are required in UI: `pinCode`, `houseNumber`, `locality`, `city`, `district`, `state`, `country`.
- For vendor profile location, these are required in UI: `pinCode`, `houseNumber`, `locality`, `city`, `district`, `state`, `country`.
- Vendor `deliveryRegion` must be one of: `25km`, `50km`, `75km`, `100km`, `150km`, `200km`, `250km`, `300km`.
- Backend should return vendor `deliveryRadius` as a number: `25`, `50`, `75`, `100`, `150`, `200`, `250`, or `300`.
- If backend also returns `deliveryRegion`, it should match the numeric radius, for example `"25km"`.

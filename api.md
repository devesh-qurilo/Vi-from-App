# Location API Contract

This frontend expects buyer and vendor location APIs to use the same address shape for GET and PUT responses.

## Shared Address Object

```json
{
  "_id": "address-id",
  "houseNumber": "233",
  "locality": "MG Road",
  "city": "Bengaluru",
  "district": "Bengaluru Urban",
  "state": "Karnataka",
  "country": "India",
  "pinCode": "560001",
  "fullAddress": "233, MG Road, Bengaluru, Bengaluru Urban, Karnataka, India, 560001",
  "latitude": 11.9716,
  "longitude": 22.5946,
  "location": {
    "type": "Point",
    "coordinates": [22.5946, 11.9716]
  },
  "createdAt": "2026-06-09T08:00:43.520Z",
  "updatedAt": "2026-06-09T14:39:52.668Z"
}
```

Important:
- `coordinates` must be `[longitude, latitude]`.
- `latitude` and `longitude` should also be returned as top-level fields for easier frontend use.
- `fullAddress` should be generated by backend if not sent from frontend.

## Validation Rules

- `houseNumber` is required.
- `latitude` is required.
- `longitude` is required.
- For buyer profile location, these are required in UI: `pinCode`, `houseNumber`, `locality`, `city`, `district`, `state`, `country`.
- For vendor profile location, these are required in UI: `pinCode`, `houseNumber`, `locality`, `city`, `district`, `state`, `country`.
- Vendor `deliveryRegion` is optional in the header modal, but required in vendor profile modal. Backend may store it as numeric `deliveryRadius`.
